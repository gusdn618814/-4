# 과제 2) 피보나치 및 GCD 시간 복잡도 프로파일링 보고서

## 1. 과제 개요
- **목표**: 재귀적 피보나치 수열 $F(n)$을 구현하고, 인접한 두 피보나치 수 $F(n)$과 $F(n-1)$의 GCD를 계산하며 시스템 자원(CPU 시간 및 메모리)의 변화를 측정한다.
- **측정 범위**: $n = 5$부터 $5$씩 증가시키며 $n = 45$까지 측정.
- **측정 항목**: 실행 시간(CPU Time), 메모리 점유 상태, 알고리즘 결과값.

## 2. 알고리즘 및 복잡도 분석 (Big-O)

### 2.1. 재귀적 피보나치 (Fibonacci)
- **구현**: $F(n) = F(n-1) + F(n-2)$ (Base case: $F(0)=0, F(1)=1$)
- **시간 복잡도**: 각 호출마다 두 개로 분기되는 재귀 트리를 형성하므로 **$O(2^n)$** 의 지수적 시간 복잡도를 가진다. (정확히는 황금비 $\phi^n$)
- **공간 복잡도**: 재귀 호출 스택의 깊이가 $n$에 비례하므로 **$O(n)$** 이 소요된다.

### 2.2. 유클리드 GCD (Greatest Common Divisor)
- **특성**: 인접한 피보나치 수 $(F_n, F_{n-1})$는 서로소($GCD=1$)이며, 이는 유클리드 알고리즘에서 가장 많은 단계를 거치는 **최악의 입력(Worst-case)**이다.
- **시간 복잡도**: 입력 숫자의 자릿수에 비례하는 **$O(\log n)$** 의 효율성을 가진다.

## 6. 결과 분석 및 결론
- **CPU 사용량**: $n$이 $30$을 넘어가면서부터 재귀 호출 횟수가 기하급수적으로 증가하여 실행 시간이 급격히 늘어나는 것을 확인하였다. 이는 $O(2^n)$ 이론과 일치한다.
- **메모리 사용량**: 재귀 호출 스택의 깊이가 깊어짐에 따라 메모리 사용량이 미세하게 증가하지만, 지수적으로 늘어나는 시간 복잡도에 비하면 상대적으로 완만하게($O(n)$) 증가함을 알 수 있다.
- **GCD 검증**: 가장 계산이 오래 걸리는 피보나치 수 쌍을 입력했음에도 불구하고, GCD 계산은 거의 즉각적으로 이루어져 유클리드 알고리즘의 효율성을 입증하였다.
