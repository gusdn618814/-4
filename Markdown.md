# 과제 2) 피보나치 및 GCD 시간 복잡도 프로파일링 보고서

## 1. 과제 개요

* **목표**: 재귀적 피보나치 수열 $F(n)$을 구현하고, 인접한 두 피보나치 수 $F(n)$과 $F(n-1)$의 GCD를 계산하면서 CPU 시간과 메모리 사용량을 분석한다.
* **측정 범위**: $n = 5$부터 1씩 증가시켜 $n = 40$까지.
* **측정 항목**: 실행 시간(CPU Time), 메모리 사용 특성, GCD 결과값.

## 2. 알고리즘 및 복잡도 분석 (Big-O)

### 2.1. 재귀적 피보나치

* **구현**: `F(n) = F(n-1) + F(n-2)` (Base case: `F(0)=0, F(1)=1`)
* **시간 복잡도**: 호출이 이진 트리 형태로 전개되므로 **$O(2^n)$** (정확히는 $\phi^n$, $\phi≈1.618$).
* **공간 복잡도**: 재귀 깊이가 $n$에 비례 → **$O(n)$**.

### 2.2. 유클리드 GCD

* **특성**: 인접 피보나치 수 $(F_n, F_{n-1})$는 서로소이므로 최악의 경우를 제공한다.
* **시간 복잡도**: 입력 크기(비트 수)에 대해 **$O(\log n)$**.

## 6. 분석 및 결론

1. **CPU 사용량**

   * $n$이 증가할수록 실행 시간이 급격히 증가하였다.
   * 이는 재귀 피보나치가 $O(2^n)$의 지수적 시간복잡도를 갖는다는 이론과 일치한다.

2. **메모리 사용량**

   * 본 실험에서는 별도의 메모리 프로파일링 도구를 사용하지 않고 이론적 분석을 기반으로 평가하였다.
   * 재귀 깊이가 $n$에 비례하므로 스택 메모리 사용량은 $O(n)$으로 증가한다.

3. **GCD**

   * 인접 피보나치 쌍에 대해 유클리드 알고리즘은 매우 빠르게 동작하였다.
   * 시간복잡도는 $O(\log n)$이며 전체 실행 시간에는 큰 영향을 미치지 않는다.

> **결론**: 재귀적으로 구현된 피보나치 함수는 $O(2^n)$의 비효율적인 시간복잡도를 가지므로 입력이 커질수록 실행 시간이 급격히 증가한다. 반면 GCD는 $O(\log n)$으로 매우 효율적이며, 전체 실행 시간은 피보나치 계산이 지배한다. 따라서 전체 알고리즘의 시간복잡도는 **$O(2^n)$**이다.
